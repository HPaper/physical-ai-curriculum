# /study-end

세션을 다음 세션·다음 기기로 안전하게 인수인계한다. **아무 상태도 Claude 컨텍스트 창에만 남기지 않는다** — 전부 파일로 옮긴다.

수행할 것:

1. **Diff 요약**:
   - `git status` 실행.
   - `git diff --stat` 실행.
   - 이번 세션에서 무엇을 바꿨는지 3–5줄로 요약해 사용자에게 보여준다.

2. **`STUDY_LOG.md` 갱신** (Edit 도구 사용):
   - `Currently studying` 블록의 `Focus`, `Last updated`를 현재 상태에 맞게 수정. 시간은 `YYYY-MM-DD HH:MM · <기기 힌트>` 형식.
   - `Recently improved` 맨 위에 이번 세션 한 줄 추가: `YYYY-MM-DD · lecNN · <무엇을 개선>`. 항목이 10개를 넘으면 가장 오래된 항목 제거.
   - **`▶ NEXT` 블록을 완전히 새로 쓴다** — 다음 세션이 (아마도 다른 기기·새 채팅) 이 블록만 붙여넣으면 정확히 이어갈 수 있게. 필수 요소:
     - 현재 어디까지 완료됐는지(구체적 파일·섹션·상태).
     - 남은 즉시 작업 1–3개(가장 자연스러운 다음 스텝).
     - 결정 대기 중인 것이 있으면 명시.
   - 새 아이디어·질문이 있으면 Backlog / Open questions에도 추가.

3. **커밋 제안**:
   - 커밋 메시지 초안을 제시한다. 스타일은 최근 커밋 참조(`git log -5 --oneline`) — 접두 없이 명확한 한국어 요약.
   - `STUDY_LOG.md`와 강의 변경사항을 **함께 하나의 커밋**에 담는다(진도와 콘텐츠가 같이 이동).
   - **사용자 승인 전에는 커밋·푸시하지 않는다.** 승인 후 `git commit` → `git push`.

4. **마무리 확인**: 커밋·푸시 완료 후 `git status`가 clean인지 확인하고, 다음 세션에서 사용할 ▶ NEXT 블록을 마지막으로 사용자에게 한 번 더 보여준다(붙여넣기 편의).
