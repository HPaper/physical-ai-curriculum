# CLAUDE.md — Physical AI 커리큘럼 학습·개선 헌법

이 리포는 **67강 커리큘럼(0–66강, 15파트)**이자, 그 위에서 사용자와 Claude가 **강의를 한 개씩 학습하며 자료를 개선**해 나가는 살아있는 학습 공간이다. 이 문서는 매 세션에 Claude가 자동으로 읽는 규약이다.

---

## 0. 세션 시작·종료 리추얼 (가장 중요)

사용자는 **여러 기기·여러 세션**에서 이 리포를 이어가며 학습한다. Claude 컨텍스트 창은 자주 리셋된다. 따라서 **모든 상태는 리포 파일에 있어야 한다** — 특히 `STUDY_LOG.md`.

### 세션 시작 시 (사용자가 `/study-start`를 부르지 않아도 첫 응답 전에)
1. `STUDY_LOG.md`를 읽는다.
2. `## Currently studying`와 `## ▶ NEXT` 블록을 사용자에게 요약해 보여준다.
3. 현재 강의 파일 경로를 확인하고(필요 시 파일을 짧게 스캔), **다음 마이크로 스텝 1–3개**를 제안한다.
4. 사용자가 선택하기 전까지 편집은 시작하지 않는다.

### 세션 종료 시 (사용자가 `/study-end`를 부르거나 명시적으로 마무리를 지시할 때)
1. 이번 세션 diff를 요약한다(`git status`, `git diff --stat`).
2. `STUDY_LOG.md`를 갱신한다:
   - `## Currently studying`의 `Last updated` 갱신.
   - `## Recently improved`에 한 줄 추가 (`YYYY-MM-DD · lecNN · 무엇을 개선`).
   - `## ▶ NEXT`를 **다음 세션이 붙여넣기만 해도 이어질 수 있는 완결된 프롬프트**로 다시 쓴다.
3. 필요하면 Backlog / Open questions도 갱신한다.
4. 커밋·푸시는 **사용자 승인 후에만** 진행한다 (커밋 메시지 초안은 제시).

리추얼 세부는 `.claude/commands/study-start.md`, `.claude/commands/study-end.md`에 슬래시 커맨드로도 등록되어 있다.

## 1. 강의 파일 규약 (기존 v3 관습을 명문화)

각 강의 `.md`는 다음 요소를 유지한다.

- **핵심 수식 3단** — 직관 → 의미 → 형식.
- **Worked example** — 손계산이 가능한 최소 사례.
- **시각자료 ≥5장** — `images/lecNN/` 아래. 인용된 모든 수치는 `images/lecNN/gen_figs.py` 실행으로 재현 가능해야 한다.
- **실행 검증 코드** — 본문 수치를 재현하는 파이썬 스니펫.
- **이전/다음 강의 네비게이션 링크** — 문서 맨 아래.
- **흔한 오해 섹션** (해당 강의에서 특히 헷갈리기 쉬운 개념이 있으면).
- **참고문헌** — 강의 말미에 목록화.

### 개선 원칙
- **독자 우선**: 정의 나열보다 "왜 필요한가"를 먼저.
- **예시 > 산문**: 문장 다섯 줄 늘릴 시간에 그림 하나·표 하나.
- **리터럴 정규화**: 인라인 코드/기호는 백틱, 수식은 `$…$`/`$$…$$`.
- **내부 §참조 동기화**: 강의 번호가 바뀌면 참조 링크도 함께 갱신.
- **양방향 링크**: 이전/다음 강의 네비는 절대 끊지 않는다.

## 2. 파일 지도

- `part01-orientation/` … `part15-frontier/` — 15개 파트, 각 파트 폴더 안에 강의 `.md` 파일.
- `images/lecNN/` — 강의별 생성 스크립트와 이미지.
- `README.md` — 커리큘럼 전체 개요·진행 상태 표.
- `STUDY_LOG.md` — **학습 진도의 단일 진실**. 다음 세션이 무엇을 할지 여기서 결정된다.
- `CLAUDE.md` — 이 문서.
- `.claude/commands/` — 세션 리추얼 슬래시 커맨드.

---

## 3. 강의 파일 안 개인 노트 규약

학습하다 떠오른 질문·아이디어는 **해당 강의 `.md` 파일 하단**에 HTML 주석으로 남긴다. 렌더링에 안 잡히고, git으로 모든 기기에 자동 전파되며, 다음에 그 강의를 편집할 때 Claude가 함께 읽는다.

```html
<!-- STUDY:
- Q: 이 정의는 [7]의 정의와 어떻게 다른가?
- 아이디어: 그림 4에 실제 궤적 오버레이 추가
- 개선: worked example의 마지막 단계 계산 재확인 필요
-->
```

별도 사이드카 파일(`_notes.md` 등)은 만들지 않는다 — "한 강의 = 한 파일" 규약을 유지하기 위함.

## 4. Claude 작업 위생

- **컨텍스트 창 아끼기**: 현재 강의와 직접 관련 없는 파일은 열지 않는다. 검증 코드는 전문이 아니라 요약을 인용.
- **작은 개선을 자주**: 강의 하나에 대해 한 번에 여러 축(그림 추가 + 오해 섹션 + 참고문헌 보강 + …)을 동시에 건드리지 말고, 한 축씩 커밋. Diff가 리뷰 가능해야 한다.
- **§참조·번호 무결성**: 강의 번호나 섹션 번호를 바꿀 때는 리포 전체에서 검색·동기화. 최근 커밋(예: `55f5a0b`)에서 확립된 관습.
- **auto memory는 로컬**: Claude Code의 `~/.claude/projects/...` 메모리는 이 Orin PC에만 남는다. 다른 기기·다른 세션에서도 필요할 지식은 반드시 이 리포의 파일(주로 CLAUDE.md, STUDY_LOG.md, 강의 내 `<!-- STUDY: -->` 주석)로 승격한다.
